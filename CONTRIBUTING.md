# Contributing

We welcome all contributions to this project! 

The following is a set of guidelines for contributing to our project. Feel free to propose changes to this document in a pull request.

## Code of Conduct

This project and everyone contributing to it is governed by our [Code of Conduct](CODE_OF_CONDUCT.md). By contributing, you are expected to uphold this code. 

## How to contribute

Following these guidelines helps maintainers understand your report, reproduce the behavior, and find related reports. Contributes and bug reports are tracked as Github issues which can be created for this project [here](https://github.com/UBC-DSCI-310-2025W2/asteroidprep/issues).

If you are creating a bug report, please include as many details as possible in order for maintainers to reproduce the behavior. The following should be included at minimum:
1. Use a clear and descriptive title for the issue.
2. Describe the exact steps to reproduce the bug.
3. Provide specific examples to demonstrate the steps. These examples should include links to files or Github projects, or code snippets that can be copy and pasted. If using snippets in the issue, use [Markdown code blocks](https://help.github.com/articles/markdown-basics/#multiple-lines). 
4. Describe the behavior you observed after following the outlined steps and explain the specific problem with this behavior.
5. Explain the expected behavior and why. 
6. Include details about your machine configuration and environment.

**Direct contributors**
- Use issues for listing and assigning tasks that need to be accomplished.
- Use the project board to organize the issues.
- Pull at the beginning of each work session, and push at the end.
- Features and fixes are to be contributed in a Github flow workflow, where branches are created for each feature or fix, which are reviewed and critiqued by at least one other teammate before the pull request is accepted.

**External contributors**
- External contributors should first open an issue to explain how they would like to help.
- If welcomed, external contributors should fork the repo and send their changes via a pull request. 

These contribution guidelines were based on [Atom's contribution guidelines](https://github.com/atom/atom/edit/master/CONTRIBUTING.md).
